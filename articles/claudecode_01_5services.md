# Claude Codeで5つのWebサービスを自動化した全記録｜失敗も含めてリアルに語る

正直に言う。最初は半信半疑だった。

「AIがコードを書いてくれる」とは聞いていたが、「事業の自動化まで任せられる」とは思っていなかった。それが今、Claude Codeなしでは開発が考えられなくなっている。

この記事では、私が実際にClaude Codeを使って構築した5つのWebサービスと、その自動化の仕組みを全公開する。綺麗な成功談だけでなく、**ハマったポイント・失敗した経緯も含めて**リアルに書く。

---

## 1. なぜClaude Codeで事業を作るのか

副業でWebサービスを作ろうとしたとき、最大のボトルネックは「時間」だった。

本業をこなしながら夜間に開発するとなると、1つのサービスを作るだけで数ヶ月かかる。しかも「コードを書く時間」より「何を書くか考える時間」のほうが長い。

Claude Codeを使い始めて変わったのは、この**「考える時間」の大部分を委譲できるようになったこと**だ。

具体的には：

- **要件を自然言語で伝えるだけで実装が進む**：「ユーザーがURLを入力したらスキャンしてスコアを返すAPI」と伝えれば、エンドポイント・バリデーション・エラーハンドリングまで書いてくれる
- **デバッグが速い**：エラーログを貼り付ければ原因と修正案が返ってくる
- **リファクタリングの提案をしてくれる**：「このコード、もっとシンプルにできますか？」と聞くだけでいい

ただし、**「全部やってくれる魔法ツール」ではない**。設計判断はこちらがする必要があるし、出力を確認・修正するリテラシーも求められる。

そのバランスを理解した上で使えば、**1人開発者の生産性が3〜5倍になる感覚**は間違いない。

## 2. サービス①〜⑤の紹介と自動化の仕組み

### サービス① GEO診断SaaS（AIによるスキャン自動化）

**概要**：URLを入力すると、そのサイトのGEO（Generative Engine Optimization）対策スコアを自動診断するWebアプリ。

**自動化した部分**：
- URLからのHTMLスクレイピング
- 構造化データ（Schema.org）の有無・内容の解析
- メタデータ（OGP・description）の評価
- AIによるコンテンツの「引用可能性」スコアリング
- 診断レポートのPDF生成

**Claude Codeへの指示例**：
```
Next.js + TypeScriptで、URLを受け取りHTMLをスクレイピングして
構造化データをパースし、GEOスコアを0-100で返すAPIを作って。
スコアリングロジックは: 構造化データ有無20点、FAQ形式10点、
メタデータ完備20点、見出し構造20点、コンテンツ明確性30点。
```

**実際に動いたもの**：約2時間で動くプロトタイプができた。通常なら1日かかる作業量。

**ハマりポイント**：スクレイピング対象サイトがCloudflareで保護されていると503が返ってきて詰まった。Claude Codeに相談したら「Puppeteerで実ブラウザ経由にするか、User-Agentを変えてリトライする実装を追加しましょう」と即提案してくれた。

---

### サービス② アフィリエイトサイト（記事自動生成・投稿）

**概要**：特定ジャンルのキーワードリストから記事を自動生成し、WordPressに自動投稿する仕組み。

**自動化した部分**：
- キーワードリストのCSV読み込み
- Claude APIを使った記事本文の生成（3,000〜5,000字）
- アイキャッチ画像の自動生成（Stable Diffusion API連携）
- WordPress REST APIへの自動投稿
- 投稿後のインデックス申請（Google Search Console API）

**技術スタック**：Python + Claude API + WordPress REST API

**Claude Codeへの指示例**：
```python
# こんな構造で作りたい
keyword_list = load_csv("keywords.csv")
for keyword in keyword_list:
    article = generate_article(keyword)  # Claude API
    image = generate_image(keyword)      # SD API
    post_to_wordpress(article, image)    # WP REST API
    request_indexing(post_url)           # GSC API
```

この擬似コードを渡したら、各関数の実装・エラーハンドリング・ログ出力まで込みで完成形を書いてくれた。

**失敗した話**：最初の200記事は品質が低くてGoogleに評価されなかった。「SEOに強い記事」のプロンプトエンジニアリングに2週間かかった。Claude Codeはコードを書くのは得意だが、**「何を生成させるか」のプロンプト設計は人間がやる必要がある**。

### サービス③ AI占いアプリ（占い文章生成）

**概要**：生年月日・名前・悩みを入力すると、AIが個別の占い文章を生成するWebアプリ。PAY.JPを使った有料課金付き。

**自動化した部分**：
- ユーザー入力の受付フォーム
- 占いロジック（数秘術・西洋占星術のルール）のプロンプト変換
- Claude APIによる占い文章の生成
- 結果のPDF出力
- PAY.JPでの決済処理

**Claude Codeへの指示例**：
```
Supabase + Next.jsで、生年月日から数秘術の数字を計算し、
それをプロンプトに含めてClaude APIで占い文章を生成するアプリを作って。
PAY.JP決済は500円で、決済完了後にのみ結果を表示する。
```

**面白かった点**：占いのロジック（数秘術の計算式、星座の割り当てロジック）をClaude Codeに「数秘術の計算ロジックを実装して」と頼むと、一般的な計算ルールをそのまま実装してくれた。ドメイン知識の実装に強い。

**収益状況**：公開1ヶ月で数十件の購入。占いというジャンルは**ファネルが短くて収益化しやすい**と実感した。

### サービス④ 不動産価格予測AI（データ自動取得・分析）

**概要**：国土交通省の公開データを定期取得・蓄積し、地域別の不動産価格トレンドを可視化・予測するWebサービス。

**自動化した部分**：
- 国土交通省API（不動産取引価格情報）からのデータ取得（定期バッチ）
- データのクレンジング・正規化
- 地域別・物件種別のトレンド分析
- Recharts.jsを使ったグラフ自動生成
- 価格予測モデルの適用（線形回帰）

**Claude Codeへの指示例**：
```
国土交通省の不動産取引価格情報APIからデータを取得して
Supabaseに保存するcronジョブを作って。
その後、都道府県・市区町村コードで絞り込んで
直近5年の価格推移をグラフ表示するページも作って。
```

**実際の開発時間**：データ取得バッチ→1時間、グラフページ→2時間、予測機能→3時間。合計6時間で動くプロトタイプ完成。

**ハマりポイント**：国交省APIのレスポンス形式が独特で、JSONの中にCSV形式のデータが入っているという二重構造だった。Claude Codeに「このAPIレスポンスをパースして」と渡したら、構造を解析して適切なパーサーを書いてくれた。自分でやったら半日かかっていた作業。

### サービス⑤ SNS自動投稿システム（note毎日自動投稿）

**概要**：テーマリストとテンプレートをもとにAIが記事を生成し、note（ブログプラットフォーム）に毎日自動投稿する仕組み。

**自動化した部分**：
- テーマの自動選択（カレンダーに基づくローテーション）
- Claude APIによる記事本文生成
- note非公式APIへの自動投稿
- 投稿後のX（Twitter）への告知ポスト
- 投稿ログのSlack通知

**技術的なポイント**：
noteは公式APIを公開していないため、Puppeteerでブラウザ操作を自動化する方法を採用した。

```python
# noteへの自動投稿のフロー
1. Claude APIで記事生成（タイトル+本文+タグ）
2. Puppeteerでnoteにログイン
3. 新規投稿フォームに内容を入力
4. 公開ボタンをクリック
5. 投稿URLを取得してSlack通知
```

このフロー全体をClaude Codeに「Puppeteerとpythonで実装して。ログインはcookieを保存して再利用する方式で」と伝えたら、セッション管理・リトライ処理込みで実装してくれた。

**運用結果**：毎朝7時に自動投稿が走って3ヶ月継続。手動作業ゼロで月30本の記事が公開されている。

## 3. Claude Codeを使ってみて分かった3つのこと

### ① 「設計」と「実装」の分離が重要

Claude Codeはコードを「書く」のは得意だが、「何を書くべきか」を決めるのは人間だ。

「〇〇を作って」と丸投げすると、動くコードは出てくるが設計が自分の意図と違うことがある。**「こういう構造にしたい、この設計の理由は〜」と伝えてから実装を依頼する**と、意図通りのコードが出てくる確率が上がる。

### ② コンテキストの管理が生産性を左右する

チャットが長くなるほど、Claude Codeの返答の精度が下がることがある（コンテキストウィンドウの限界）。

実践的な対処法：
- **機能ごとに新しいチャットを開く**
- 長いファイルは「このファイルの〇〇行目から〇〇行目の部分を修正して」と範囲指定する
- 複雑な変更は「まず設計を確認してから実装」と2段階にする

### ③ エラーの説明が上手い

デバッグでの真価はエラー解析だ。エラーログをそのまま貼り付けると：

1. エラーの原因を特定
2. なぜそのエラーが起きるかの説明
3. 修正コード
4. 再発防止のための改善提案

この4つを一度に返してくれる。**「なぜ」を教えてくれることで、同じミスを繰り返さなくなる**のが地味に大きい。

## 4. ハマりやすいポイントと解決策

実際に開発していて詰まった場面と、その解決策をまとめる。

### ハマり①：「動くコードが出てくるけど、自分で直せない」

**原因**：Claude Codeが生成したコードを理解せずに使い、エラーが出たときに対処できない。

**解決策**：コードを受け取ったら「このコードの構造を日本語で説明して」と必ず聞く。理解した上で使うことで、修正・拡張が自分でできるようになる。

### ハマり②：「最初の設計が間違っていてあとで詰む」

**原因**：小さい機能から積み上げていくうちに、全体のアーキテクチャが歪になる。

**解決策**：開発前に「このサービスのDB設計・APIエンドポイント・フォルダ構成を提案して」と一度設計レビューをしてもらう。後から直すより最初に整える方が10倍速い。

### ハマり③：「外部APIの仕様変更に追随できない」

**原因**：Claude Codeの学習データには古いAPIドキュメントが含まれることがある。

**解決策**：公式ドキュメントのURLをコンテキストに含めるか、最新のドキュメントをコピーして「この仕様で実装して」と伝える。

### ハマり④：「テストコードを書いてもらったが、実環境では動かない」

**原因**：モックが多すぎて実際の動作を確認できていない。

**解決策**：「ユニットテストより、実際にAPIを叩く統合テストを優先して書いて」と伝える。特に副業開発では網羅的なテストより、核心部分の動作確認が重要。

## 5. まとめ・次のステップ

Claude Codeで5つのサービスを作って実感したのは、**「アイデアと実行の距離」が劇的に縮まった**ということだ。

以前は「これを作るには3ヶ月かかる」と諦めていたものが、「週末にプロトタイプを作って検証→続けるか判断」というサイクルで動けるようになった。

副業・個人開発でClaude Codeを活用するためのステップを整理すると：

1. **小さく始める**：まず1つの機能だけ作ってみる
2. **設計から依頼する**：実装の前に構造を相談する
3. **コードを理解する習慣をつける**：ブラックボックスにしない
4. **自動化できる部分を特定する**：繰り返し作業をClaude Codeに任せる
5. **失敗を恐れない**：プロトタイプは壊して学ぶためにある

Claude Codeは「コードを書くツール」ではなく、**「事業を作るパートナー」**として使うのが正解だと思っている。
